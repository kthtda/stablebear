# masspcf bug-scan findings

Confirmed correctness defects found by a broad automated API scan on 2026-06-07. A fleet of agents exercised the public surface across 15 functional areas (CPU and CUDA paths; an RTX 4070 Ti was present) and tried to break it; every candidate was then independently re-run by an adversarial verifier that re-derived the correct answer and classified it. Only verifier-confirmed defects are listed here.

**Nothing here is fixed.** Each confirmed bug has a *failing* regression test asserting the correct behavior, so the suite is red until the bug is fixed. Usability gaps (not correctness bugs) live in `friction-triage.md`.

## Summary

- **53 confirmed bugs**: 4 critical, 22 high, 16 medium, 11 low.
- 2 candidates rejected as expected/documented behavior (listed at the end).
- Tests: `test/python/test_bugscan_<area>.py` (73 tests / 76 parametrized items, all red).

### Running the tests

```bash
cmake --install cmake-build-debug        # tests import the INSTALLED package
cd test && python -m pytest python/test_bugscan_*.py -q   # all red until fixed
```
Hard crashes (SIGSEGV/SIGABRT/SIGFPE) are exercised in a subprocess via `test/python/_bugscan_support.py` so they cannot abort the pytest session.
